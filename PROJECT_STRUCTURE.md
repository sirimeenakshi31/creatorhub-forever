## Conventions

### Routing
- File-based via `src/routes/`. Add a file, get a route.
- Server endpoints live under `src/routes/api/`.
- Never edit `src/routeTree.gen.ts` — it's regenerated by the Vite plugin.

### Adding a new tool
1. Add an entry to `TOOLS` in `src/lib/tools.ts` with `slug`, `kind`, and
   `systemPrompt` (for text tools).
2. Add the slug to `systemForSlug` in `src/lib/server-utils.ts`.
3. The generic dispatcher (`tools.$slug.tsx`) picks it up automatically.
4. For custom UI, create `src/routes/tools.<slug>.tsx` instead.

### Server endpoints
- Always validate input length (Zod or manual).
- Always call `rateLimit(request, key, max, windowMs)` first.
- Always return CORS headers and JSON via the local `json()` helper.
- Fall back to mock output on missing keys — never throw a 500 for a missing
  env var without a clear message.

### Design tokens
- Defined in `src/styles.css` using `oklch()` for HDR-ready colors.
- Use semantic classes (`bg-background`, `text-foreground`, `bg-primary`),
  never raw `bg-white` / `text-black`.

### Server runtime
- Production target is Cloudflare Workers (nodejs_compat).
- Avoid Node-only packages (sharp, canvas, child_process). Use fetch + Web APIs.
